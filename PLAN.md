# MVP магазина приложений на Android
Список эпиков и задач в рамках эпика:
## Подготовка проекта - 2 дня
- Создать проект и создать все необходимые папки для удобства, чтобы сразу начать делать в рамках чистой Архитектуры - 2 ч
- Подключить все зависимости - retrofit, okhttp, navigation, coil и так далее - 2 ч
- Создавть все необходимые базовые модели в домейн слое, настроить DI, создать репозитории- 2 ч
- Настроить навигацию между экранами. Пока что с моковыми композабл функциями чтобы потом просто заменить функцию на настоящую - 3 ч
## Начальный экран: список приложений - 3 дня

- Подключить бэкенд (создать dto, мапперы, interceptor, настроить создание инстанса апи с помощью DI)-5 ч
- Реализовать методы получения списка приложений и получения приложения по id - 1 ч
- Накидать базовый дизайн для экрана - 3 ч
- Настроить UIState, добавить ViewModel и связать ее с репозиторием, написать функцию для получения списка приложений - 2 ч
- Сверстать экран, связать с ViewModel, реализовать отоброжение state - загрузка, ошибка, список приложений - 6 ч

## Карточка приложения - 2 дня

- Накидать базовый дизайн для экрана - 3 ч
- Сверстать экран связать с ViewModel, реализовать отоброжение state - загрузка, ошибка, список приложений  - 6 ч
- Реализовать навигацию к экрану карточки , то есть передача id приложения из списка на экран карточки - 2 ч
- Протестировать правильный переход на экран карточки - 1 ч

## Установка и удаление приложения - 4 дня

- Подключить все необходимое для управления скачиванием апк приложений - 2 ч
- Реализовать бизнес логику для установки (проверка достаточно ли места, проверка интернета, платное или бесплатное приложение, прерывание скачивания) - 6 ч
- Бизнес логика для удаления (проверка можно ли удалить данное приложение, уточнение у пользователя хочет ли он удалить) - 3 ч
- Реализовать состояние скачивания - 2 ч
- оповещение о успешной установке или удалении. оповещение об ошибке - 2 ч
- Связать всю логику с presentation слоем - 4 ч
- Протестировать все основные сценарии и исправление багов - рабочий день
